# 跨区域变更指南

D3KeyHelper 没有传统前后端分层，但一个用户功能经常同时影响 AHK 自动执行段、GUI 控件、INI 持久化、动态热键、游戏检测和发布文档。本目录用于防止只修改其中一个位置。

## 可用指南

| 文档 | 何时使用 |
|---|---|
| [变更影响检查表](./change-impact-checklist.md) | 新增设置、助手、热键、像素检测、Profile 行为或发布字段 |

## 使用原则

1. 修改前先用 `rg` 搜索字段名、控件变量、标签、函数和默认值。
2. 沿“用户输入 → GUI/INI → 共享状态 → 动作 → 清理/保存”的完整路径检查。
3. 以 `src/d3keyhelper.ahk`、`scripts/build.ps1`、`.github/workflows/build.yml` 和 `README.md` 的实际实现为依据，不套用 Web/服务端模板。
4. 发现新的高风险联动时，把它补充到检查表，而不是只留在提交说明中。
